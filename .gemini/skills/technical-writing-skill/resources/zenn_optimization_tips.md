# Zenn Optimization & SEO Tips

技術記事を「届ける」ための、メタデータとSEOの戦略ガイドです。

## 1. 絵文字（Emoji）の戦略的選択
Zennの絵文字は、記事の「第一印象」を決定します。

- **技術スタックに合わせる**: 
    - React/Next.js: ⚛️, ⚡️, 🚀
    - Rust: 🦀, ⚙️
    - Security: 🛡️, 🔑
    - AI/LLM: 🤖, 🧠
- **感情に合わせる**:
    - 苦労話/ハマりポイント: 😰, 🛠️, 🌋
    - 爆速/効率化: 🏃‍♂️, 💨, 📈
- **差別化**: 
    - トレンドに乗りたい時はあえて目立つ色（🍎, 🐯, 🌈）を使うのも手です。

## 2. スラッグ（Slug）の設計
URLになるスラッグは、短く、内容を推測しやすく、SEOを意識したものにします。

- **Good**: `nextjs-app-router-auth`, `rust-ownership-tutorial`
- **Bad**: `article-123`, `my-thought-today`
- **ルール**: 半角英数字とハイフンのみを使用。日本語は含めない。

## 3. トピック（Topics）の選定
適切なタグ付けは、Zennのアルゴリズムに記事を正しく認識させるために不可欠です。

- **コア技術を必ず入れる**: `React`, `TypeScript`, `Next.js` など。
- **抽象的なトピックも混ぜる**: `Architecture`, `Testing`, `productivity` など。
- **最大5つ**: Zennでは5つまで設定可能。人気タグ（Zennのトップページで確認）を1つは混ぜるのがセオリー。

## 4. 検索エンジン最適化 (SEO)
- **H2/H3見出しにキーワードを入れる**: 「結論」だけでなく「React Server Componentsのデータフェッチの結論」のように具体的に。
- **序文に結論を含める**: Google検索結果のスニペットに表示されることを意識する。
- **画像にalt属性を設定する**: `![Next.jsの認証フロー図](url)`

## 5. 公開後の拡散戦略
- **X (Twitter) での投稿**: 
    - 「技術的な結論」＋「記事へのリンク」＋「アイキャッチ画像（Zennが生成）」の構成で。
    - 著者の「熱量」が伝わる一言を添える。
- **GitHubスターとの連携**: 記事内で紹介したコードがGitHubにある場合、そのリポジトリへのリンクを目立つ場所に配置する。
- **定期的なリライト**: 技術が古くなったら「[2026年版]」のように更新し、記事を腐らせない。
