# Technical Blog Writing Skill

このスキルは、ZennやQiitaなどのプラットフォームで、単に情報を伝えるだけでなく、読者の心を動かし、トレンド入りするような「体温のある」技術記事を執筆するためのガイドです。

## 🧠 Mindset: 脱AI・人間味の極意

AI生成記事の最大の特徴は「正解を、平均的に、無機質に述べること」です。これに対抗し、人間にしか書けない価値を提供するためのマインドセットを持ちましょう。

-   **感情を隠さない**: 「便利だ」という事実だけでなく、「これを見つけた時、椅子から転げ落ちそうになった」という驚きや、「3時間溶かして絶望した」という苦悩を隠さず記述する。
-   **主観を貫く**: 客観的な正解はAIに任せ、あなたは「私はこう思う」「これが好きだ」「これは汚いと感じる」という強い主観（スタンス）を表明する。
-   **「隙」を作る**: 完璧な教科書を目指さない。失敗談、迷い、個人的なこだわりなどの「人間らしい隙」が、読者の共感とコメントを誘発する。
-   **読者を「お前」や「戦友」として見る**: 画面の向こうにいる、かつての自分と同じ悩みを抱えるエンジニアに直接語りかける。

## ✍️ Workflow

### 1. 執筆前の「魂」のセットアップ
-   **誰の、どの瞬間の感情を救うか？**: 「React初心者」ではなく「深夜2時に型定義が通らず、PCを閉じようとしているReact初心者」くらい具体的に設定する。
-   **自分の「原体験」を掘り起こす**: その技術に触れたきっかけ、最初に感じた違和感、解決した瞬間の脳内麻薬。

### 2. 構成（型を破るための型）
-   **導入 (Hook)**: 読者の「あるある」を突く。AIのような「近年、AIの進化は目覚ましく…」といった退屈な一般論から始めない。
-   **結論の先出し (Value)**: 読者の時間を尊重し、最速で価値を渡す。ただし、そこに至る「物語」を予感させる。
-   **本文 (The Journey)**: 成功体験だけでなく、あえて「遠回りした過程」を記述する。
-   **おまけ・ポエム (Soul)**: 本筋に関係ない個人的なこだわりや、技術に対する愛を語るセクションを設ける。

### 3. ドラフト（脱AIライティング術）
-   **リズムを崩す**: 短文と長文を混ぜる。「〜です。〜ます。」の連続を避け、倒置法や体言止め、独り言を混ぜて会話のリズムを作る。
-   **比喩とユーモア**: 専門用語を、日常的なものや文化的な文脈（アニメ、映画、歴史など）で例える。
-   **コードに「意図」を込める**: 綺麗なコードを載せるだけでなく、「なぜこう書いたのか？」「ここを端折るとどうなるか？」という行間の意図を語る。

## 🚀 Anti-AI Viral Checklist

公開前に、以下の「AIっぽさ」を排除できているか確認してください。

-   [ ] **タイトルに「毒」や「愛」があるか？**: 優等生なタイトルになっていないか。
-   [ ] **冒頭3行で「人間」が書いていると分かるか？**: 著者の顔や状況が見えるか。
-   [ ] **「個人的には〜」という主観が3箇所以上あるか？**: 統計データではなく、あなたの意見が含まれているか。
-   [ ] **AIなら絶対に書かない「無駄な一言」があるか？**: ユーモア、皮肉、情熱的な叫びなど。
-   [ ] **読者に問いかけているか？**: 一方的な講義ではなく、対話になっているか。
