# Story Structure Evaluation

## Framework Selection Guide

Match the presentation's purpose to the optimal narrative framework:

| Purpose | Best Framework | Why |
|---------|---------------|-----|
| Proposal / Persuasion | PREP | Leads with conclusion, backs with evidence |
| Problem solving | Problem-Solution | Pain-driven, shows clear fix |
| Report / Status share | SDS | Bookend summaries aid retention |
| Transformation / DX | AS-IS → TO-BE | Gap visualization motivates change |
| Vision / Kickoff | Why → What → How | Inspires before instructing |
| Decision support | Comparative Analysis | Structured options lead to choice |

## Framework Definitions & Expected Slide Arcs

### 1. PREP (Point-Reason-Example-Point)

```
[Title] → [Conclusion/Point] → [Reason 1] → [Reason 2] → [Example/Evidence] → [Restate Point] → [CTA] → [Ending]
```

**Expected slides**: 8–10
**Key check**: Does slide 2 state the conclusion upfront?
**Common failure**: Burying the conclusion at the end (report style, not PREP).

### 2. Problem-Solution

```
[Title] → [Problem description] → [Impact/Urgency] → [Root cause] → [Solution] → [Implementation plan] → [Expected results] → [CTA] → [Ending]
```

**Expected slides**: 9–11
**Key check**: Is the problem emotionally compelling? Does solution directly address root cause?
**Common failure**: Weak problem statement that doesn't create urgency.

### 3. SDS (Summary-Details-Summary)

```
[Title] → [Executive Summary (top 3)] → [Detail 1] → [Detail 2] → [Detail 3] → [Reinforcing Summary] → [Next Steps] → [Ending]
```

**Expected slides**: 8–10
**Key check**: Can you skip to first and last summary slides and get the full picture?
**Common failure**: Details don't map back to summary points.

### 4. AS-IS → TO-BE

```
[Title] → [Current state (AS-IS)] → [Problems with current] → [Future vision (TO-BE)] → [Gap analysis] → [Transition plan] → [Timeline] → [CTA] → [Ending]
```

**Expected slides**: 9–11
**Key check**: Is the contrast between AS-IS and TO-BE dramatic and specific?
**Common failure**: TO-BE is vague ("better system") instead of concrete ("30% faster processing").

### 5. Why → What → How (Golden Circle)

```
[Title] → [Why (purpose/values)] → [What (goal/vision)] → [How (execution plan)] → [Key milestones] → [Team/Resources] → [CTA] → [Ending]
```

**Expected slides**: 7–9
**Key check**: Does "Why" create emotional resonance before "What" introduces logistics?
**Common failure**: Starting with "What" or "How" — skipping the inspirational "Why".

### 6. Comparative Analysis

```
[Title] → [Background/Context] → [Evaluation criteria] → [Option A] → [Option B] → [Option C] → [Comparison table] → [Recommendation] → [Ending]
```

**Expected slides**: 8–10
**Key check**: Are criteria defined BEFORE options are presented? Is recommendation justified?
**Common failure**: Presenting options without clear evaluation framework.

## Story Structure Scoring Criteria

### Title Test (Slide Titles Only)
Read ONLY the slide titles in sequence. Score:
- **10**: Titles alone tell a complete, persuasive story
- **7**: Titles convey the main flow but miss some connections
- **5**: Titles are a mix of messages and topic labels
- **3**: Titles are mostly generic labels ("Background", "Data")
- **1**: Titles are meaningless or absent

### 空雨傘 (Sky-Rain-Umbrella) Test
For persuasive presentations, verify all three elements:
- **空 (Sky/Fact)**: Objective data/situation presented (e.g., "市場シェアが15%低下")
- **雨 (Rain/Interpretation)**: Meaning derived from facts (e.g., "このままでは3年で撤退圏内に")
- **傘 (Umbrella/Action)**: Concrete proposal (e.g., "新チャネルへの投資で回復を図る")

Missing any element weakens the persuasion:
- No 空: Claims without evidence → "Why should I believe you?"
- No 雨: Data without insight → "So what?"
- No 傘: Analysis without action → "What do you want me to do?"

### Transition Coherence
Check slide-to-slide flow:
- Each slide should logically follow the previous one
- No abrupt topic jumps without section breaks
- Conclusion connects back to opening hook/problem

### So What? Test
For EVERY slide, ask "だから何？" (So what?):
- If you cannot answer in one sentence, the slide lacks a clear message
- If the answer isn't relevant to the overall story, the slide may be unnecessary

## Slide Title Transformation Guide

Convert topic labels to message-driven titles:

| NG (Topic Label) | OK (Message/Assertion) |
|-------------------|----------------------|
| 売上実績 | Q3売上は目標比120%を達成 |
| 背景 | 市場シェアの低下が3四半期連続で進行中 |
| 課題 | レガシーシステムが月40時間の手作業を発生させている |
| 提案 | クラウド移行で運用コストを年間30%削減できる |
| まとめ | 3つの施策で来期の目標達成が見込める |
| スケジュール | 3フェーズ6ヶ月で段階的に移行する |
| チーム体制 | 専任5名+兼任3名の少数精鋭で推進 |
| リスク | 移行期間中のダウンタイムを最小化する方策がある |

**Rule**: Every title should be readable as a headline — it makes a claim or states a finding.
