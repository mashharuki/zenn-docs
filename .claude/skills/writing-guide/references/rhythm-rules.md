# リズム層のAI臭ルール

出典: coji 氏「AI臭は語彙よりリズムに出る」（natural-japanese スキルの実測分析、
7つのLLMモデル・406本のコーパスを人間の文章と比較）。
語彙の禁止リストだけでは消えないAI臭の「本体」を扱う。

## なぜリズムなのか

禁止語を全部置換しても「なんかAIっぽい」が残るのは、不自然さが単語ではなく
文章全体の統計的な癖に宿っているから。読者は文長の揺れや段落の呼吸を
無意識に感知している。実測では、文長リズムの均質さはどのLLMでも55〜95%の文書で検出され、
最も普遍的なAI臭だった。

## 検出ルールと直し方

### 1. 文長リズムの均質さ（low burstiness）

**症状**: 平均的な長さの文（実測例では平均56±22.5モーラ）が淡々と並ぶ。
グラフにすると文長がほぼ横一線。

**直し方**: 短文を打ち込む。「違う。」「ここでハマった。」のような数文字の文は
人間の文章の呼吸そのもの。逆に、関連する2文を1つに結合して長い文を作ってもいい。
大事なのは分散を作ることで、平均を変えることではない。

### 2. 段落構造の均質さ

**症状**: 複数の段落がきっかり同じ文数（例: 全部4文）で刻まれる。

**直し方**: 内容の重さに段落の長さを従わせる。強調したい一言は1文だけの段落に独立させる。
説明が必要な箇所は長い段落を許す。

### 3. 体言止めゼロ

**症状**: 文末がすべて「です・ます・だ・である」等の述語で終わり、名詞で止まる文が1つもない。

**背景**: 「体言止めの多用がAI臭」という通説は実測で反証されている。
人間コーパスでは書き手の約60%が体言止めを使うのに対し、AIはほぼ0%。
つまり体言止めの「欠如」こそがAIの指紋。

**直し方**: 効く場所に数回だけ体言止めを置く。ただし連続・均一
（毎段落の締めが全部名詞止め）は逆方向のAI臭になるので、あくまで散らす。
ゼロでも全部でもなく「たまに」が人間のリズム。

### 4. 「〜ではなく」反復

**症状**: 「AではなくB」という否定→肯定の対比構文が繰り返し登場する。
AIは論点を際立たせるためにこの構造を過剰に使う。

**直し方**: 対比が本当に必要か問い直す。多くの場合、否定をやめて
「Bだ」と直接言う方が力強い。1記事に数回までが目安。

### 5. 翻訳調（品詞列の不自然さ）

**症状**: 英語の構文が透けて見える品詞の並び。「〜することが可能です」
「〜という点において」「〜を提供します」など、文法的には正しいが
日本語として流暢でない。

**直し方**: 声に出して読む。口頭で人に説明するときの言い方に直す。
「提供します」→「使えます」、「〜という点において」→「〜では」。

## 修正ループの回し方（重要）

AIは「ここを直せ」と指摘されると、その箇所を**別の方向に整えてしまう**性質がある。
指摘を全部機械的に修正すると、指標だけ改善して不自然さが移動する（Goodhart の法則）。

だから検出結果は禁止事項ではなく「疑いの提示」として扱い、このループを回す。

1. `scripts/ai_smell_lint.py` で疑い箇所を列挙する
2. 一箇所ずつ、文脈の中で「本当に不自然か」を自分で判断する。意図的な表現なら残す
3. 直すと決めた箇所だけ修正する
4. 再 lint して収束を確認する。修正が新しい癖（体言止めの入れすぎ、短文の乱発）を
   生んでいたら、そこも同じ手順で判断する

2〜3周で収束しない場合、局所修正ではなく段落ごと書き直した方が早い。

## 過剰矯正リスト（人間の言葉を狩らない）

実測で「人間の方がよく使う」と判明している表現。AI臭リストに入れてはいけない。

| 表現 | 実測 |
| --- | --- |
| 「最後に」 | 人間48回に対しAI 2回。人間の日常語 |
| 「まさに」 | 人間24回に対しAI 0回。使ってよい |
| 文頭の反復（アナフォラ） | 人間93%に対しAI 41%。人間のリズム技法 |
