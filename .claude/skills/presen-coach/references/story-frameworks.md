# Story Frameworks — ナラティブ構造とスライド設計

## フレームワーク選択ガイド

```
目的が「説得・提案」?
  → Yes: 承認が必要? → Yes: PREP法 / 問題解決型
               → No (行動喚起): Why-What-How型
  → No: 報告・共有? → Yes: SDS法 / AS-IS TO-BE型
         教育・説明? → Yes: 基礎→応用型
         比較検討?   → Yes: 比較分析型
```

---

## 1. PREP 法（提案・説得の王道）

**Point → Reason → Example → Point**

| スライド | 内容 | 狙い |
|---|---|---|
| 1 | タイトル | 期待感 |
| 2 | **結論（Point）** | 最重要メッセージを先出し |
| 3 | 現状の課題 | 「なぜ必要か」の文脈 |
| 4–5 | **理由・根拠（Reason）** | なぜその結論が正しいか |
| 6–7 | **事例・データ（Example）** | 具体的な証拠 |
| 8 | **結論の再提示（Point）** | メッセージの強化 |
| 9 | ネクストアクション | 行動への誘導 |

**使い所**: 経営承認・新規提案・投資判断

---

## 2. 問題解決型（課題起点の提案）

**Problem → Cause → Solution → Result**

| スライド | 内容 |
|---|---|
| 1 | タイトル |
| 2 | アジェンダ |
| 3 | **現状の問題** — 数値や事実で示す |
| 4 | **問題の原因** — なぜ起きているか |
| 5 | **解決策の概要** |
| 6–7 | **解決策の詳細** — How to |
| 8 | **期待される効果** — Before / After |
| 9 | 実施計画・スケジュール |
| 10 | まとめ＋承認依頼 |

**使い所**: 業務改善提案・システム導入・コスト削減提案

---

## 3. SDS 法（報告・共有）

**Summary → Details → Summary**

| スライド | 内容 |
|---|---|
| 1 | タイトル |
| 2 | **サマリー（結論3点）** |
| 3–7 | **詳細** — 各トピックを深掘り |
| 8 | **サマリー再掲** — 結論を強化 |
| 9 | Q&A・ディスカッション |

**使い所**: 月次報告・プロジェクト進捗・調査結果共有

---

## 4. AS-IS → TO-BE 型（変革・改善系）

| スライド | 内容 |
|---|---|
| 1 | タイトル |
| 2 | 全体像（変革の概要） |
| 3–4 | **AS-IS（現状）** — 事実・課題・痛み |
| 5 | **Gap（課題の本質）** — なぜこのままではいけないか |
| 6–7 | **TO-BE（理想）** — ビジョン・目標 |
| 8–9 | **移行計画** — ロードマップ |
| 10 | まとめ＋コール |

**使い所**: 組織変革・DX推進・戦略転換

---

## 5. Why → What → How 型（ビジョン提示）

Simon Sinek の "Golden Circle" に基づく。

| スライド | 内容 |
|---|---|
| 1 | タイトル |
| 2 | **Why** — なぜこれが重要か（価値観・目的） |
| 3 | **What** — 何を実現するか（目標・成果） |
| 4–6 | **How** — どうやって実現するか |
| 7 | **Call to Action** |

**使い所**: キックオフ・ビジョン発表・採用説明会

---

## 6. 比較分析型（意思決定支援）

| スライド | 内容 |
|---|---|
| 1 | タイトル |
| 2 | 比較の背景・前提条件 |
| 3 | **評価軸の定義** |
| 4–5 | **各選択肢の説明** |
| 6 | **比較表** |
| 7 | **推奨案と根拠** |
| 8 | ネクストアクション |

**使い所**: ツール選定・ベンダー比較・戦略オプション提示

---

## スライドタイプ × コンテンツの対応表

| コンテンツ | 最適なスライドタイプ | 避けるべき |
|---|---|---|
| 単一の強いメッセージ | リード（大文字中央） | 箇条書きで埋める |
| 手順・プロセス | ステップリスト / 横矢印 | 長文箇条書き |
| 数値・実績 | 大きな数字 + ラベル | 表にまとめる |
| 2つの比較 | 2カラム（左右対称） | 混在した箇条書き |
| 3〜4の選択肢 | カード3枚並び | リスト |
| 時間軸・ロードマップ | タイムライン（横） | 箇条書き |
| コード・技術詳細 | ダーク背景コードブロック | スクリーンショット |
| 引用・声 | ブロッククォート + 人物 | 本文に混ぜる |

---

## スライドタイトルの書き方

タイトルは「トピック名」ではなく「メッセージ（結論）」で書く。

| NG（トピック型） | OK（メッセージ型） |
|---|---|
| 「売上実績」 | 「Q3売上は目標比120%を達成」 |
| 「現状の課題」 | 「手作業が月50時間の無駄を生んでいる」 |
| 「提案内容」 | 「自動化で工数を80%削減できる」 |
| 「まとめ」 | 「3つの施策で来期利益を2倍にする」 |

---

## 1スライドの情報量チェック

- **箇条書き**: 最大5点、1点12語以内
- **発話時間**: 1スライドで話す内容は30秒〜1分が目安
- **文字数目安**: 1スライド全体で150文字以内（タイトル除く）
- **画像・図**: 1スライドに1点が原則（多くても2点）

詰め込みになっているスライドは分割を推奨：
- 「AとBについて」→ スライドを2枚に分ける
- 箇条書きが7点以上 → 上位3点＋「詳細は別途」に絞る
